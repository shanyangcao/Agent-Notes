# Prompt 基础概念

- ### 经典 CRCF 框架  

  **角色**（Role）

  **任务**（Task）

  **上下文背景**（Context）

  **输出格式**（Format）

  ### 常见的Prompt扩展要素

  除了 CRCF，很多高级 Prompt 还会包含以下要素，以进一步提升输出质量：

  **示例**：

  提供 1-2 个你期望的输出样例，而且放在提示词的开头、角色前、任务后、用户输入前／后等）也会显著影响模型性能

  **思维链**（cot）：

   引导 AI 由大化小，一步步写出步骤，现在很多模型都有cot

  **约束**

  明确 AI 不能做什么，或者必须遵守的规则，比如 “不要使用专业术语”、“字数限制在 500 字以内”

  **目标**

   说明你希望达成的最终效果，比如 “帮助新手快速理解项目”、“生成可直接上线的代码”  

  ### 示例

  ```
  【角色】你是一位经验丰富的前端架构师，擅长分析和文档化项目。
  【任务】请帮我分析我提供的Vue项目代码，理解其整体结构和业务逻辑，并生成一份清晰的项目说明文档。
  【上下文】这是一个电商网站的前端项目，使用Vue3 + Vite + Element Plus构建，代码位于 /src 目录下。
  【约束】文档语言要通俗易懂，避免使用过于专业的术语，重点突出核心业务流程。
  【输出格式】输出一份Markdown文档，包含项目概览、目录结构解析、核心业务逻辑和注意事项四个部分。
  ```

  除此之外还有zero-shot和few-shot
